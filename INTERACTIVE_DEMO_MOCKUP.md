## 🎯 卡片設計細節

### 單張卡片結構

```html
┌──────────────────────────────────────────────┐
│                                              │
│  [Favicon]  標題文字（粗體、灰色 900）        │
│   (40x40)   URL（小字、灰色 500）             │
│                                              │
└──────────────────────────────────────────────┘
   ↑ 白色背景、圓角 12px、陰影、邊框
```

### 樣式規格

- **背景：** `#ffffff`（白色）
- **邊框：** `#e5e5e5`（預設）→ `#ff507a/30`（hover）
- **陰影：** `shadow-md`（預設）→ `shadow-lg`（hover）
- **圓角：** `12px`（rounded-xl）
- **Padding：** `16px`（p-4）
- **游標：** `grab`（預設）→ `grabbing`（拖曳中）

### 拖曳狀態樣式

- **Ghost（跟隨滑鼠）：** `opacity: 0.5` + 輕微旋轉（2deg）
- **插入線：** 粉紅色虛線（`#ff507a`, 2px, dashed）
- **其他卡片：** 平滑移動（transition 200ms ease-out）

## 🛠️ 技術實作

### 推薦方案：SortableJS

**為什麼選擇 SortableJS？**

✅ **優點：**
- 同時支援 Desktop（滑鼠）和 Mobile（觸控）
- 動畫流暢（60 FPS）
- API 簡潔易用
- 輕量（~10KB gzipped）
- 支援鍵盤操作（無障礙）
- 跨瀏覽器相容

❌ **缺點：**
- 需引入外部庫（但體積小）

### 簡易實作範例

```html
<!-- HTML -->
<div id="demo-cards" class="space-y-3">
  <div class="demo-card">
    <img src="react-icon.png" alt="React" class="w-10 h-10">
    <div>
      <div class="font-medium text-gray-900">React - Official Docs</div>
      <div class="text-sm text-gray-500">react.dev</div>
    </div>
  </div>
  <!-- 其他卡片 -->
</div>

<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/npm/sortablejs@latest/Sortable.min.js"></script>
<script>
  new Sortable(document.getElementById('demo-cards'), {
    animation: 200,
    easing: 'ease-out',
    ghostClass: 'opacity-50',
    chosenClass: 'shadow-xl',
  });
</script>
```

## 📱 響應式設計

### Desktop（1024px+）

- 卡片寬度：`max-width: 28rem`（448px）
- 滑鼠拖曳
- Hover 效果明顯

### Tablet（768px - 1023px）

- 卡片寬度：`max-width: 24rem`（384px）
- 觸控或滑鼠拖曳
- Hover 效果保留

### Mobile（320px - 767px）

- 卡片寬度：`100%`（填滿容器）
- 觸控拖曳
- 長按 200ms 開始拖曳（避免與捲動衝突）
- Hover 效果移除

---

## ♿ 無障礙支援

### 鍵盤操作

- `Tab` - 聚焦到下一張卡片
- `Shift + Tab` - 聚焦到上一張卡片
- `Space` 或 `Enter` - 選取卡片（進入拖曳模式）
- `↑` - 向上移動卡片
- `↓` - 向下移動卡片
- `Space` 或 `Enter` - 放下卡片
- `Esc` - 取消拖曳

### ARIA 標記

```html
<div role="list" aria-label="可拖曳的示範卡片">
  <div role="listitem"
       tabindex="0"
       aria-grabbed="false"
       aria-label="React 官方文檔">
    <!-- Card content -->
  </div>
</div>
```

### 螢幕閱讀器提示

- 卡片聚焦時：「React 官方文檔，可拖曳項目，第 1 項，共 5 項」
- 選取時：「已選取 React 官方文檔，使用方向鍵移動」
- 移動時：「React 官方文檔移動至第 3 項」
- 放下時：「React 官方文檔已放置於第 3 項」

## 📊 效能影響

### 評估

- **庫大小：** SortableJS ~10KB gzipped
- **動畫：** CSS transform（GPU 加速，60 FPS）
- **卡片數量：** 僅 5 張（對效能無影響）
- **Lighthouse 影響：** 預估 < 3 分（可接受）

### 優化策略

1. **延遲載入** - 使用 Intersection Observer，卷軸到 demo 區塊才初始化
2. **CDN 快取** - 使用 jsDelivr CDN（全球節點）
3. **Passive listeners** - 觸控事件使用 passive mode

## 🎬 可選：引導動畫

首次載入時，可播放一次自動拖曳示範（3 秒）：

```
1. 第 1 張卡片自動提起（1s）
2. 移動到第 3 個位置（1s）
3. 放下並恢復（1s）
4. 提示文字：「試試自己拖曳看看！」
```

**優點：**
- 降低學習曲線
- 吸引注意力

**缺點：**
- 可能干擾使用者
- 增加複雜度

**建議：** 可選功能，預設關閉，使用 `?demo=auto` 參數開啟
